# Hot Compare — GitHub 项目横向对比

用户指定 2-5 个 GitHub 项目，按优化版对比方法论进行深度分析，一步到位输出 HTML 报告、信息卡片和科普素材。

---

## 触发方式

用户输入格式示例：
- 「对比 A vs B vs C」
- 「compare owner1/repo1 owner2/repo2」
- 「横向对比 https://github.com/A/B 和 C/D」
- 「项目对比：vuejs/core reactjs/react angular/angular」

> **对比对象数量**：2-5 个。少于 2 个→建议用 hot-analysis。多于 5 个→建议分批。

---

## 执行步骤

### 步骤 1：解析对比对象列表

从用户输入中提取所有 `owner/repo`：
- URL 格式：`https://github.com/owner/repo` → 提取 owner, repo
- 短格式：`owner/repo` → 直接使用
- 支持的分隔符：`vs` / `和` / `对比` / 逗号 / 空格混用

### 步骤 2：并行获取数据

对每个对比对象：
- **元数据**：运行 `python scripts/fetch_repo_info.py owner/repo`
- **实际内容**：用 web_fetch 获取 README.md（GitHub raw URL）
- 等待所有并发请求完成后进入下一步

### 步骤 3：AI 按优化版 5 模块方法论撰写对比分析

> 完整方法论详见 `reference/对比方法论_优化版.md`

**模块一：单项独立深解**

每个对比对象，逐项分析以下 **7 项**（比原方法论增加 1 项）：
1. 基础档案（简介/开发者/社区/更新时间/协议/热度）
2. 核心功能（能做什么/不能做什么/独有能力）
3. ⭐ **最简上手路径**（一行命令 + 第一个操作验证 + 老手知道的捷径）
4. 使用方式（接入/部署/操作流程/配置复杂度）
5. 环境与依赖（系统/软件/硬件/网络）
6. 使用成本（免费/付费/资源占用）
7. 适用场景与人群

**模块二：全维度矩阵对比 + 决策指引**

10 行标准对比表格（比原方法论增加 1 行）：
1. 核心定位 · 2.功能差异 · 3.上手难度 · 4.环境依赖 · 5.社区维护
6. 适用场景 · 7.独家优势 · 8.短板限制 · 9.兼容性 · 10.⭐ **学习曲线**

每行末尾附带 **决策短句**，例如：「如果你上手时间≤1 天 → 选 A」

**模块三：🆕 真实场景对抗**

3 个典型场景，展示各项目的实际表现：
- 场景 1：快速原型（谁最快出活）
- 场景 2：生产部署（谁最稳定可靠）
- 场景 3：复杂定制（谁最灵活可扩）
- 每个场景：「用项目 A 怎么做」vs「用项目 B 怎么做」vs「用项目 C 怎么做」

**模块四：组合串联分析**

1. 原生兼容性总判（是否有官方集成/API 对接）
2. 原生组合方案（如适用）
3. ⭐ **非兼容串联方案**：即使无法原生对接，给出"工具 A 完成 → 输出 → 作为工具 B 输入"的实际串联路径
4. 每套方案的完整步骤 + 分工明细 + 配置要点

**模块五：🆕 高手进阶路径**

每个项目独立给出：
- 入门信号（做到什么算入门）
- 会用信号（做到什么算会用）
- 精通信号（做到什么算精通）
- 下一步学习资源（官方文档章节/社区/教程）
- CheatSheet（3 条最常用命令/操作，一页速查）

### 步骤 4：生成双输出

**输出 A：对比型 HTML 报告**
- 文件名：`output/compare_项目1_vs_项目2_YYYYMMDD_HHMM.html`
- 设计：深色极简 · 顶部 5 模块锚点导航 · 对比表格 · 折叠面板 · 回到顶部

**输出 B：对比型卡片集**
- 文件名：`output/compare_项目1_vs_项目2_cards_YYYYMMDD_HHMM.html`
- 设计：宽度 1080px，高度自适应 · 对比总览卡 + 各项目身份卡 + 组合方案卡 + CheatSheet 卡

### 步骤 5：直接触发素材生成

分析完成后直接进入素材生成，无需用户二次指令：
- 生成综合封面提示词（融合所有对比项目的视觉元素）
- 为对比卡片集中的**每一张卡片**生成对应的口播解说文案（150-200 字/段，理性介绍风格，每段聚焦该卡片主题）
- 口播文案命名格式：`【口播文案】/01_卡片主题_script.md`，与卡片一一对应
- 素材目录：`output/素材输出_YYYYMMDD_HHMM/`

### 步骤 6：向用户汇报

- 对比了哪几个项目
- 核心差异一句话
- 双输出路径 + 素材路径

## 设计规范

共用 hot-analysis 的设计体系（Taste-Skill + UI UX PRO MAX）：
- 深色极简 · 青色+靛蓝 · 玻璃拟态 · 渐变标题
- HTML 交互：锚点导航 · 对比表格 · 折叠面板 · 回到顶部 · 响应式
- 卡片：1080px 宽 · 高度自适应 · 紧凑不留白

---

## 注意事项

- 所有分析内容以 README.md 和官方文档为唯一依据
- README 未记载的信息，标注「文档未记载，暂无可靠信息」
- 严禁编造功能、流程、搭配方式
- 技术术语附带通俗解释
- 对比对象≥3 时，表格和流程图保持清晰，维度不缺失
