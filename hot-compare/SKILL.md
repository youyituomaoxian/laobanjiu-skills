# Hot Compare — GitHub 项目横向对比

用户指定 2-5 个 GitHub 项目，按优化版对比方法论进行深度分析，一步到位输出 HTML 报告、信息卡片和科普素材。

---

## 触发方式

用户输入格式示例：
- 「对比 A vs B vs C」
- 「compare owner1/repo1 owner2/repo2」
- 「横向对比 https://github.com/A/B 和 C/D」
- 「项目对比：vuejs/core reactjs/react angular/angular」

> **对比对象数量**：2-5 个。少于 2 个→建议用 hot-analysis。多于 5 个→建议分批。

---

## 执行步骤

### 步骤 1：解析对比对象列表

从用户输入中提取所有 `owner/repo`：
- URL 格式：`https://github.com/owner/repo` → 提取 owner, repo
- 短格式：`owner/repo` → 直接使用
- 支持的分隔符：`vs` / `和` / `对比` / 逗号 / 空格混用

### 步骤 2：并行获取数据

对每个对比对象：
- **元数据**：运行 `python scripts/fetch_repo_info.py owner/repo`
- **实际内容**：用 web_fetch 获取 README.md（GitHub raw URL）
- 等待所有并发请求完成后进入下一步

### 步骤 3：AI 按优化版 5 模块方法论撰写对比分析

> 完整方法论详见 `reference/对比方法论_优化版.md`

**模块一：单项独立深解**

每个对比对象，逐项分析以下 **7 项**（比原方法论增加 1 项）：
1. 基础档案（简介/开发者/社区/更新时间/协议/热度）
2. 核心功能（能做什么/不能做什么/独有能力）
3. ⭐ **最简上手路径**（一行命令 + 第一个操作验证 + 老手知道的捷径）
4. 使用方式（接入/部署/操作流程/配置复杂度）
5. 环境与依赖（系统/软件/硬件/网络）
6. 使用成本（免费/付费/资源占用）
7. 适用场景与人群

**模块二：全维度矩阵对比 + 决策指引**

10 行标准对比表格（比原方法论增加 1 行）：
1. 核心定位 · 2.功能差异 · 3.上手难度 · 4.环境依赖 · 5.社区维护
6. 适用场景 · 7.独家优势 · 8.短板限制 · 9.兼容性 · 10.⭐ **学习曲线**

每行末尾附带 **决策短句**，例如：「如果你上手时间≤1 天 → 选 A」

**模块三：🆕 真实场景对抗**

3 个典型场景，展示各项目的实际表现：
- 场景 1：快速原型（谁最快出活）
- 场景 2：生产部署（谁最稳定可靠）
- 场景 3：复杂定制（谁最灵活可扩）
- 每个场景：「用项目 A 怎么做」vs「用项目 B 怎么做」vs「用项目 C 怎么做」

**模块四：组合串联分析**

1. 原生兼容性总判（是否有官方集成/API 对接）
2. 原生组合方案（如适用）
3. ⭐ **非兼容串联方案**：即使无法原生对接，给出"工具 A 完成 → 输出 → 作为工具 B 输入"的实际串联路径
4. 每套方案的完整步骤 + 分工明细 + 配置要点

**模块五：🆕 高手进阶路径**

每个项目独立给出：
- 入门信号（做到什么算入门）
- 会用信号（做到什么算会用）
- 精通信号（做到什么算精通）
- 下一步学习资源（官方文档章节/社区/教程）
- CheatSheet（3 条最常用命令/操作，一页速查）

### 步骤 4：生成三输出

**输出 A：对比型 HTML 报告**
- 文件名：`output/compare_项目1_vs_项目2_YYYYMMDD_HHMM.html`
- 设计：深色极简 · 顶部 5 模块锚点导航 · 对比表格 · 折叠面板 · 回到顶部

**输出 B：海报级卡片 HTML → PNG**
- HTML 文件：`output/compare_项目1_vs_项目2_卡片_YYYYMMDD_HHMM.html`
- **固定画布 1080×810 (4:3)**，不使用响应式布局
- 每张卡片独立成海报：主标题 + 副标题 + 内容层 + 底部数据
- 生成 HTML 后**必须运行** `python scripts/export_posters.py <html_path>` 导出 PNG
- 海报卡片编号与内容对应：

| 卡 # | 内容 | 口播对应 |
|------|------|---------|
| 1 | **封面总览**：标题 + 各项目 Stars/定位 + 一句话互补关系 | 开场引入 |
| 2 | **各项目身份卡**（每个项目 1 张）：核心特性 + 安装命令 + ✅适合/❌不含 + ⏱时间 | 逐一介绍 |
| 3 | **矩阵对决**：5-10 维对比表 + 3 个真实场景对抗 + 决策总结 | 横向对比 |
| 4 | **组合串联**：三阶流水线 / 组合方案 + 各步输出 + 时间 | 串联方案 |
| 5 | **高手路径**：入门/精通标签 + CheatSheet 命令 | 收尾指引 |

> 卡片张数根据对比对象数量调整（2 个→约 5 张，3 个→6-7 张，5 个→约 8 张）。每张卡片内容覆盖对应口播段。

### 步骤 5：直接触发素材生成

分析完成后直接进入素材生成，无需用户二次指令：

1. **创建素材目录** `output/素材输出_YYYYMMDD_HHMM/`，含子文件夹：
   - `【封面提示词】/` `【口播文案】/` `【项目提示词】/` `【HTML报告】/` `【海报PNG】/`
2. **复制产出物**：
   - 长页 HTML + 卡片 HTML → `【HTML报告】/`
   - 6 张海报 PNG → `【海报PNG】/`
3. **生成内容**：
   - 综合封面提示词（融合所有对比项目的视觉元素）
   - 各项目独立生图提示词（9:16 竖版，英文）
   - **一篇完整流畅的口播稿**（500-800 字），按卡片内容分为对应段落，每个段落之间用自然过渡句衔接，不用"第一张卡、第二张卡"这种分段标记。语言口语化但保持理性，像在给朋友讲解三个工具的区别和用法。段落对应关系（与海报卡片编号一致）：卡1（封面总览）→ 卡2+（各项目身份卡）→ 矩阵对决卡 → 组合串联卡 → 高手路径卡
   - 口播命名格式：`【口播文案】/00_完整口播稿_script.md`

### 步骤 6：向用户汇报

- 对比了哪几个项目
- 核心差异一句话
- 三输出路径（长页 + 卡片 HTML + 卡片 PNG）+ 素材路径

## 设计规范

> 仅适用于海报卡片 HTML。长页报告使用原有的响应式设计体系。

### 画布
- **固定 1080×810 (4:3)**，无响应式，禁用 `clamp()`
- 每张卡片视为独立海报，内容撑满版面

### 字体层级（px 固定值）

| 层级 | 尺寸 | 用途 |
|------|------|------|
| 封面标题 | 90px | 卡片 1 主标题 |
| 身份卡标题 | 72px | 各项目单独卡片主标题 |
| 小标题 | 56px | 矩阵/流水线标题 |
| 副标题 | 24px | 标题下方引导语 |
| 正文 | 18px | `.txt`、描述段落 |
| 卡内标题 | 16px | `.bx .h` 内部框标题 |
| 正文小 | 14px | `.bx .b`、`table`、`.pi` pills |
| 数据大字 | 36px | `.num` Stars 等数字 |
| 小标签 | 13px | `.lbl`、`.label` |

### 颜色（十六进制，确保 WCAG AA）
- Strong: `#e2e8f0` · Body: `#94a3b8` · Muted: `#64748b`
- 渐变标题: `#f1f5f9 → #22d3ee / #fbbf24 / #818cf8`
- **禁止** `rgba(255,255,255,.xx)` 透明度写法

### CSS 约束
```css
.poster{width:1080px;aspect-ratio:4/3;padding:28px 32px;display:flex;flex-direction:column;}
.poster > *:not(.noise):not(.deco){position:relative;z-index:3;}
.deco{position:absolute;z-index:0;}
.ftr{margin-top:auto;}
```

### 海报 PNG 导出
```bash
python scripts/export_posters.py output/xxx_卡片.html
```
→ `posters/` 子目录，2160×1620px @2x (1080×810 逻辑)。

## 长页报告设计规范
- 深色极简 · 青色+靛蓝 · 玻璃拟态 · 渐变标题
- HTML 交互：锚点导航 · 对比表格 · 折叠面板 · 回到顶部 · 响应式

---

## 注意事项

- 所有分析内容以 README.md 和官方文档为唯一依据
- README 未记载的信息，标注「文档未记载，暂无可靠信息」
- 严禁编造功能、流程、搭配方式
- 技术术语附带通俗解释
- 对比对象≥3 时，表格和流程图保持清晰，维度不缺失
