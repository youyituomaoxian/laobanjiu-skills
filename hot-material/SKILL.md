# 热门素材生成

## 触发方式
当用户说「生成素材」「热门素材」「生成科普素材」时执行本技能。
也可在「热门分析」完成后自动询问用户是否继续生成素材。

## 执行步骤

### 1. 运行 Python 辅助脚本
```bash
cd I:\cherry claw\workspaceA\Data\Agents\Github && python scripts/generate_material.py
```
脚本会自动：
- 读取最新一期 `output/latest.html`
- 解析 3 个项目的名称、Star、描述等元数据
- 在 `output/` 下创建 `素材输出_YYYYMMDD_HHMM/` 目录
- 创建子目录：【封面提示词】、【HTML报告】、【海报PNG】、【项目提示词】、【口播文案】
- 复制 HTML 报告到 `【HTML报告】/`，复制海报 PNG 到 `【海报PNG】/`
- 输出 `_projects.json`（AI 读取用）

### 2. 读取 _projects.json
找到生成的素材目录，读取 `_projects.json` 获取项目列表。

### 3. 生成综合封面提示词
- 文件名：`【封面提示词】/00_综合封面_prompt.md`
- 语言：英文
- 长度：150-250 词
- 比例：9:16 竖版
- 内容：将本批所有项目的视觉元素融合到一张图中
- 构图建议：中心主视觉 + 环绕 3 个项目子卡片 / 各项目 logo + Star 数字 + 统一的科技感背景

### 4. 为每个项目生成 2 种素材
#### ① GPT-image2 生图提示词
- 文件名：`【项目提示词】/01_项目名_prompt.md`
- 语言：英文
- 长度：100-200 词
- 风格：科普图解/信息卡片风，适合 GPT-image2 生图
- **生图比例固定 9:16 竖版（适合手机短视频/Reels）**
- 内容要素：
  - 视觉风格描述（扁平信息图 / 科技插画 / 卡片式排版）
  - 核心构图（中心元素 + 环绕信息点）
  - 数据可视化元素（Star 数字、对比图表等）
  - 配色建议（青蓝主色调、暖色强调）
  - 文字排版建议（中英双语标题 + 要点）

#### ② 口播文案
- 文件名：`【口播文案】/01_项目名_script.md`
- 语言：中文
- 长度：250-300 字
- **风格：理性介绍，直述事实，去掉情绪化开头和口语化段子**
- **固定结构**：
  1. 「今天介绍的开源项目是 **项目名**，一个 一句话定位，由 开发者 开发。」
  2. 核心功能介绍（1-2 句，说清楚干什么用）
  3. GitHub 数据（Stars、Fork、语言）
  4. 使用方法和典型场景
  5. 一句话收尾

### 5. 补充 00_本期概览.md
完善 `00_本期概览.md`，填入本期内容概要。

### 6. 向用户汇报
- 素材目录路径
- 生成了几个项目的提示词和口播文案
- 各项目名称

## 输出目录结构
```
output/素材输出_YYYYMMDD_HHMM/
├── 00_本期概览.md
├── _projects.json                        ← AI 读取用
├── 【封面提示词】/
│   └── 00_综合封面_prompt.md             ← 全项目综合生图提示词
├── 【HTML报告】/
│   ├── latest.html                     ← 最新分析报告
│   └── 海报卡片_4v3.html               ← 海报卡片（如有）
├── 【海报PNG】/
│   ├── card_01_xxx.png                 ← 2160×1620px @2x
│   ├── card_02_xxx.png
│   └── ...
├── 【项目提示词】/
│   ├── 01_项目名_prompt.md
│   ├── 02_项目名_prompt.md
│   └── 03_项目名_prompt.md
└── 【口播文案】/
    ├── 01_项目名_script.md                ← 理性介绍风格
    ├── 02_项目名_script.md
    └── 03_项目名_script.md
```

## 注意
- 素材内容（提示词和口播文案）由 AI 根据项目分析数据创作，无法模板化
- 脚本只负责目录结构和数据提取，不生成内容
- 口播文案严格控制在 300 字以内，理性介绍风格，禁情绪化开头
- 生图提示词用英文，口播文案用中文
